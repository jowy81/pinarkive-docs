**Site:** https://docs.pinarkive.com  
**Report date:** 2025-03  
**Scope:** Technical SEO, documentation SEO, crawlability, metadata, sitemap, structured data, and monitoring.

---

## 1. Summary of Changes

| Area | Status | Notes |
|------|--------|--------|
| **Site structure & crawlability** | Done | Static URLs per page; all links are HTML `<a href>`. No hash-only or client-only routing. |
| **Metadata & on-page SEO** | Done | Unique descriptive titles and meta descriptions for every doc page; OG/twitter use page-level title/description; headings and keywords aligned. |

## 2. Sitemaps

| Sitemap | URL | Purpose |
|---------|-----|--------|
| Main docs sitemap | https://docs.pinarkive.com/sitemap.xml | All documentation pages; includes `<lastmod>`, `<changefreq>`, `<priority>`. |

**Regenerate sitemap:**

- After adding/removing doc pages: run `npm run sitemap`.
- Sitemap is also regenerated after `npm run build` (postbuild script).

**Submit to Google Search Console:**

1. Open [Google Search Console](https://search.google.com/search-console).
2. Add property `https://docs.pinarkive.com` (or use a domain property).
3. **Sitemaps** → Add sitemap: `https://docs.pinarkive.com/sitemap.xml`.
4. Check coverage and sitemap status in the report.

## 3. International SEO (Future)

- **Current:** Single language (English). No hreflang.
- **When adding translations:**
  - Use language-specific URLs (e.g. `/en/`, `/es/`).
  - Add `<link rel="alternate" hreflang="en" href="https://docs.pinarkive.com/en/..." />` (and equivalent for each language) on each page.
  - Set canonical per localized version (do not canonicalize all languages to one).
  - Generate a sitemap per language and a sitemap index that references them.
  - Run a hreflang validator (e.g. [Screaming Frog](https://www.screamingfrog.co.uk/seo-spider/) or [hreflang tags testing tool](https://technicalseo.com/tools/hreflang/)) and fix any missing or inconsistent tags.

---

## 4. Canonical & Hreflang Validation

- **Canonical:** Each page has a canonical URL set via Nextra/next-seo from `theme.config.jsx` (`useNextSeoProps()` with `canonical` from current path). No cross-version duplicate content; single locale only.
- **Hreflang:** Not applicable (single language). When multilingual, validate that every translated page has correct hreflang and that the default (e.g. `x-default`) points to the preferred version.

## 6. Checklist (Pre-Release & After Doc Updates)

- [ ] Run `npm run sitemap` (or rely on postbuild) so sitemap includes new/changed pages and correct `lastmod`.
- [ ] Confirm every new doc page has unique `title` and `description` in frontmatter (no generic "Overview" or "Introduction" only).
- [ ] Confirm no new content is only behind tabs/accordions without equivalent crawlable HTML (or add crawlable fallback).
- [ ] If you added images: ensure all have descriptive `alt` attributes (e.g. `![PinArkive gateway flow](/img/diagram.png)` or `<img alt="...">`). No decorative images without `alt=""`.
- [ ] In Google Search Console: check Coverage and Sitemaps for errors; request indexing for important new URLs if needed.
- [ ] If you added translations: add hreflang, update sitemap index, and run a hreflang validator.

---

## 7. Re-Run Indexing and Monitoring After New Releases

1. **Deploy** the updated docs (including new sitemap if you ran `npm run sitemap` or postbuild).
2. **Search Console:** Use “URL Inspection” → “Request indexing” for critical new or updated pages (optional; crawling will pick them up from sitemap).
3. **Monitor:** Over the next days, check Coverage and Sitemaps for 404s, exclusions, or sitemap errors.
4. **Staging:** Ensure staging/preview domains are not submitted to Search Console and use `noindex` (e.g. `<meta name="robots" content="noindex">` or env-based) so they are not indexed.
