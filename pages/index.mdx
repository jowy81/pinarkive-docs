# Pinarkive Documentation

Upload, pin and manage IPFS content using clusters and gateways.

## Try Pinarkive in 30 seconds

Upload your first file with a single request:

```bash
curl -X POST https://api.pinarkive.com/v3/files \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -F file=@image.png
```

Replace `YOUR_API_KEY` with your API key and `image.png` with your file path. The response includes a **CID** you can use to access the file via IPFS gateways.

[Get started →](/quickstart/introduction) · [Upload your first file →](/quickstart/upload-first-file)

## API Reference

REST API for uploading files, pinning by CID, listing uploads, and removing pins. All endpoints use **API v3**.

| Endpoint | Description |
|----------|-------------|
| [POST /files](/api-reference/post-files) | Upload a file (multipart) |
| [POST /files/pin/:cid](/api-reference/post-pin) | Pin an existing CID |
| [GET /users/me/uploads](/api-reference/get-files) | List your uploads |
| [DELETE /files/remove/:cid](/api-reference/delete-file) | Remove a pin |
| [GET /users/me/clusters](/api-reference/get-clusters) | List clusters and gateways |

[View API Reference →](/api-reference/post-files)
