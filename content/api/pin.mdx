# POST /pin

Pin an existing IPFS content by its CID. Use this when you already have a CID (e.g. from another source) and want Pinarkive to pin and serve it.

## Endpoint

```
POST https://api.pinarkive.com/v3/pin
```

## Headers

| Header | Required | Description |
|--------|----------|-------------|
| `Authorization` | Yes | `Bearer YOUR_API_KEY` |
| `Content-Type` | Yes | `application/json` |

## Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `cid` | string | Yes | IPFS content identifier (CID) to pin |
| `cluster` | string | No | Target cluster (e.g. `global`, `eu`, `na`). Default: `global` |
| `expires_at` | string | No | ISO 8601 timestamp when the pin expires (timelock) |

## Example request

```bash
curl -X POST https://api.pinarkive.com/v3/pin \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"cid": "bafybeigdyrzt5sfp7udm7hu76uh7y26nf3efuylqabf3oclgtqy55fbzdi"}'
```

With cluster and expiration:

```bash
curl -X POST https://api.pinarkive.com/v3/pin \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"cid": "bafybeigdyrzt5sfp7udm7hu76uh7y26nf3efuylqabf3oclgtqy55fbzdi", "cluster": "eu", "expires_at": "2026-03-11T12:00:00Z"}'
```

## Example response

**Success (200)**

```json
{
  "cid": "bafybeigdyrzt5sfp7udm7hu76uh7y26nf3efuylqabf3oclgtqy55fbzdi",
  "cluster": "eu",
  "expires_at": "2026-03-11T12:00:00Z"
}
```

| Field | Type | Description |
|-------|------|-------------|
| `cid` | string | The pinned CID |
| `cluster` | string | Cluster where the content was pinned |
| `expires_at` | string \| null | Expiration time (ISO 8601), or null if no timelock |

**Error (4xx/5xx)**

```json
{
  "error": "Bad Request",
  "message": "Invalid CID"
}
```

## Gateway URL

After pinning, content is available at:

```
https://gateway.pinarkive.com/ipfs/<cid>
```

Use the cluster-specific gateway if you pinned to a region. See [Gateways](/concepts/gateways).
