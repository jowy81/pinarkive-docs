# DELETE /file/:cid

Remove a pin from your account by CID. The content may remain on the IPFS network if other nodes have it; this endpoint only stops Pinarkive from pinning it for you.

## Endpoint

```
DELETE https://api.pinarkive.com/v3/file/:cid
```

## Headers

| Header | Required | Description |
|--------|----------|-------------|
| `Authorization` | Yes | `Bearer YOUR_API_KEY` |

## Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `cid` | string | Yes | IPFS content identifier (path parameter) |

## Example request

```bash
curl -X DELETE https://api.pinarkive.com/v3/file/bafybeigdyrzt5sfp7udm7hu76uh7y26nf3efuylqabf3oclgtqy55fbzdi \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Example response

**Success (200)**

```json
{
  "cid": "bafybeigdyrzt5sfp7udm7hu76uh7y26nf3efuylqabf3oclgtqy55fbzdi",
  "removed": true
}
```

| Field | Type | Description |
|-------|------|-------------|
| `cid` | string | The CID that was unpinned |
| `removed` | boolean | Whether the pin was removed |

**Error (4xx/5xx)**

```json
{
  "error": "Not Found",
  "message": "Pin not found for this CID"
}
```

```json
{
  "error": "Bad Request",
  "message": "Invalid CID"
}
```

## Notes

- Only pins belonging to your account can be removed.
- After removal, gateway URLs for this CID may stop resolving from Pinarkive; the content can still be available via other IPFS gateways if it exists elsewhere on the network.
