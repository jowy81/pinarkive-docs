# Authentication

The Pinarkive API uses **Bearer token** authentication. Include your API key in the `Authorization` header for every request.

## Getting an API key

1. Sign in at [pinarkive.com](https://pinarkive.com).
2. Open your account or project settings.
3. Create or copy an API key.

Keep your API key secret. Do not commit it to version control or expose it in client-side code.

## Using your API key

Send the key in the `Authorization` header:

```bash
curl -X POST https://api.pinarkive.com/v3/upload \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -F file=@myfile.png
```

In code:

```javascript
const response = await fetch('https://api.pinarkive.com/v3/upload', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${process.env.PINARKIVE_API_KEY}`,
  },
  body: formData,
})
```

## Scopes and permissions

API keys can be restricted by scope (e.g. upload-only, read-only). Check your dashboard for the permissions attached to each key.

## Errors

- **401 Unauthorized** — Missing or invalid API key. Check the header and key value.
- **403 Forbidden** — Key is valid but not allowed for this operation.

For more details, see the [API Reference](/api/upload).
