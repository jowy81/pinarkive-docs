# Quickstart

The fastest way to upload a file to Pinarkive is with a single `curl` request.

## Upload a file

```bash
curl -X POST https://api.pinarkive.com/v3/upload \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -F file=@image.png
```

Replace `YOUR_API_KEY` with your [API key](/getting-started/authentication) and `image.png` with your file path.

## Example response

```json
{
  "cid": "bafy...",
  "cluster": "global",
  "expires_at": "2026-03-11T12:00:00Z"
}
```

- **cid** — Content identifier on IPFS. Use it to retrieve or pin the file.
- **cluster** — The cluster where the file was pinned (e.g. `global`, `eu`, `na`).
- **expires_at** — Optional; when the pin expires (if you set a timelock).

## Gateway usage

After uploading, you can access your file via IPFS gateways.

**Primary gateway**

```
https://gateway.pinarkive.com/ipfs/<cid>
```

**Secondary gateway (public IPFS)**

```
https://ipfs.io/ipfs/<cid>
```

Replace `<cid>` with the `cid` from the upload response. For cluster-specific gateways, see [Clusters](/concepts/clusters) and [Gateways](/concepts/gateways).
