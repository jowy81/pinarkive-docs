# Pinarkive Documentation

Upload, pin and manage IPFS content using clusters and gateways.

## Quickstart

Upload your first file with a single request. All API requests use the **v3** base URL and require authentication (Bearer token or API key).

```bash
curl -X POST https://api.pinarkive.com/v3/files \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -F file=@image.png
```

Replace `YOUR_API_KEY` with your [API key](/getting-started/upload-first-file#authentication) and `image.png` with your file path. The response includes a **CID** (content identifier) you can use to access the file via IPFS gateways.

[Get started →](/getting-started/introduction) · [Upload your first file →](/getting-started/upload-first-file)

## API Reference

REST API for uploading files, pinning by CID, listing uploads, and removing pins. All endpoints use **API v3**.

| Endpoint | Description |
|----------|-------------|
| [POST /files](/api/post-files) | Upload a file (multipart) |
| [POST /files/pin/:cid](/api/post-pin) | Pin an existing CID |
| [GET /users/me/uploads](/api/get-files) | List your uploads |
| [DELETE /files/remove/:cid](/api/delete-file) | Remove a pin |
| [GET /users/me/clusters](/api/get-clusters) | List clusters and gateways |

[View API Reference →](/api/post-files)
