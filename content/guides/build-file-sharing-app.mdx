# Build a File Sharing App

This guide walks you through building a simple file sharing app: users upload files, get a link, and optionally set an expiration time.

## Overview

1. **Upload** — User selects a file; your backend or client calls Pinarkive `POST /upload`.
2. **Link** — You get a `cid` back; the shareable link is `https://gateway.pinarkive.com/ipfs/<cid>`.
3. **Optional expiration** — Send `expires_at` with the upload for a timelock.

## Step 1: Backend upload endpoint

Keep your API key on the server. Expose an endpoint that:

- Accepts a file (e.g. `multipart/form-data`).
- Optionally accepts `cluster` and `expires_at`.
- Forwards the request to `POST https://api.pinarkive.com/v3/upload` with `Authorization: Bearer YOUR_API_KEY`.
- Returns the JSON response (e.g. `{ cid, cluster, expires_at }`).

See [POST /upload](/api/upload) and [Express API Template](/templates/express-api) for details.

## Step 2: Frontend

- Provide a file input or drag-and-drop component.
- On submit, send the file to your backend upload endpoint.
- When the response returns, build the gateway URL and show it (and optionally a “Copy link” button).

Example gateway URL:

```text
https://gateway.pinarkive.com/ipfs/<cid>
```

## Step 3: Optional expiration

- Add a date/time or “expires in” control in the UI.
- Send the chosen time as `expires_at` (ISO 8601) with the upload.
- Inform the user that the link will stop working after that time (see [Timelocks](/concepts/timelocks)).

## Step 4: List and delete (optional)

- Use [GET /files](/api/files) to show the user their uploads.
- Use [DELETE /file/:cid](/api/delete) to let them remove a pin and revoke the link.
